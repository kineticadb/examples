<h1>
A simple illustration of continuously updated materialized views in Kinetica
</h1>
Kinetica's vectorized processing engine makes it possible to execute complex queries on real time data. This workbook illustrates how to set up an always on analytical pipeline by chaining together a series of materialized views, where the changes cascade down from one view to another.

### Try it yourself
All the steps and instructions are provided within the workbook itself. All you need to do is follow the instructions [here](https://github.com/kineticadb/examples#how-to-run-these-examples) to load the workbook into Kinetica and try this out on your own. 

Please follow the [Install Kinetica](https://github.com/kineticadb/examples#install-kinetica), if you don't have an instance of Kinetica available.
